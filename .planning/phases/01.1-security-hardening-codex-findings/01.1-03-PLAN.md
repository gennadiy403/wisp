<objective>
Fix HIGH+MEDIUM SEC-02 findings в health-monitor daemon thread (govori.py:2497-2526).

**Делегируется Codex** через `codex exec` — race condition в daemon thread с разделяемым state, нужен careful lock-discipline.

**Two issues:**

1. **HIGH — Висящий recording при revoked Accessibility:** Когда CGEventTap disabled, fn-up event не приходит (system input заморожен). Health-monitor сейчас просто показывает HUD, но `recording` остаётся True, `audio_stream` живой, mic пишет в `audio_chunks` бесконечно. Privacy + resource leak.

2. **MEDIUM — Recovery race с unrelated HUD state:** На recovery (`elif enabled and tap_was_disabled`) безусловный `set_hud(False)` может скрыть HUD который был выставлен другим путём (например, error_retryable от API timeout, или active recording если его кто-то начал параллельно).

**Что делать (Codex):**
1. На detection revoked (`if not enabled and not tap_was_disabled`):
   - Под `_state_lock`: cancel any active recording (как cancel_recording() делает — see existing fn cancel code), close audio_stream, clear note/predict/auto-send flags, set cancelled=True
   - Только потом set_hud(error_fatal, accessibility_revoked)
2. На recovery (`elif enabled and tap_was_disabled`):
   - Перед set_hud(False) — проверить `_hud_error_mode == 'error_fatal'` И что HUD reason всё ещё accessibility (нужно либо tracking variable типа `_hud_owner = 'health_monitor'`, либо проверка на конкретный tooltip key)
   - Если HUD уже не наш — оставить как есть, только сбросить tap_was_disabled
3. Print 'Accessibility restored.' можно оставить безусловным
