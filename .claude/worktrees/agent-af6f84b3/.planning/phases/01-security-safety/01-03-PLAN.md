**2. Create `_tap_health_check(tap)` function** near `install_monitor()`:

```python
def _tap_health_check(tap):
    """Poll CGEventTap health every 7s. Daemon thread -- exits with main process."""
    tap_was_disabled = False
    while True:
        time.sleep(7)
        try:
            enabled = Quartz.CGEventTapIsEnabled(tap)
        except Exception:
            continue  # if tap reference becomes invalid, keep polling
        if not enabled and not tap_was_disabled:
            tap_was_disabled = True
            # Attempt re-enable per D-10
            try:
                Quartz.CGEventTapEnable(tap, True)
            except Exception:
                pass
            set_hud(True, mode="error_fatal", tooltip=_tooltip("accessibility_revoked"))
            print("! Accessibility revoked -- attempting re-enable", flush=True)
        elif not enabled and tap_was_disabled:
            # Still disabled -- try re-enable again
            try:
                Quartz.CGEventTapEnable(tap, True)
            except Exception:
                pass
        elif enabled and tap_was_disabled:
            # Recovered! Clear error per D-11
            tap_was_disabled = False
            set_hud(False)
            print("Accessibility restored.", flush=True)
        # If enabled and was not disabled -- normal state, do nothing
```

Key design points per locked decisions:
- D-09: Polls every 7 seconds (within the 5-10s range)
- D-10: On tap disabled, shows red HUD + tooltip, attempts CGEventTapEnable(tap, True)
- D-11: Govori continues running, does not exit. Auto-recovers on next successful check
- D-07: Uses error_fatal mode (non-retryable -- user must fix in System Settings)
- Per Pitfall 2: CGEventTapEnable may succeed temporarily but macOS re-disables. The loop handles this by checking again on the next cycle. Only clears error HUD when tap stays enabled across a full check
- Per Pitfall 3: set_hud() already dispatches to main queue internally, so calling from daemon thread is safe

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| macOS Accessibility API -> Application | Permission state can change at any time |
| PortAudio/sounddevice -> Application | Hardware availability can change at any time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 | Denial of Service | CGEventTap revoked at runtime | mitigate | Health check thread polls every 7s, attempts re-enable, shows HUD error. App continues running (SEC-02, D-09 through D-11) |
| T-01-10 | Denial of Service | System input freeze from dead tap | mitigate | CGEventTapEnable(tap, True) attempts recovery. User sees tooltip directing them to System Settings. App does NOT hold onto a dead tap that blocks input |
| T-01-11 | Denial of Service | No microphone available | mitigate | Startup warning + recording-time error with HUD feedback. App continues running (REL-01, D-12 through D-14) |
| T-01-12 | Spoofing | False "recovered" state from CGEventTapEnable | accept | macOS may re-disable tap after our re-enable. Mitigated by re-checking on next 7s cycle. Per Pitfall 2, error is only cleared when tap stays enabled across a full check cycle |
</threat_model>
